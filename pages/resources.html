The 6 Questions AI Needs You To Answer

1. PROBLEM ‚Äî "What's the pain?"
Be specific about what's broken today.

2. SUCCESS ‚Äî "What does done look like?"
How will you know it's working?

3. BOUNDARIES ‚Äî "What's NOT included?"
This stops scope creep. Define phase 2 later.

4. PIECES ‚Äî "What are the parts?"
Break it into smaller chunks.

5. ORDER ‚Äî "What must work first?"
Number them. Build foundations before features.

6. TEST ‚Äî "How do we prove it works?"
For each piece, what's a specific example?

When Things Go Wrong

Because they will. Here's how to recover.

"It's not working and I don't know why"

Stop. Read the error message. Ask AI: "What does this error mean?" Start there.

"I added a feature and now everything's broken"

Undo the last change. Git helps here. If no Git, start fresh from your last working version.

"AI keeps giving me different answers"

Be more specific. Give more context. Share what you've already tried and why it failed.

"I'm too deep to start over but it's a mess"

Stop adding features. Stabilize what you have. Sometimes "done" beats "perfect."

"AI forgot what we agreed earlier"

This is normal. Keep a doc of decisions. Paste the relevant context when starting new chats.

"I'm frustrated and want to quit"

Step away. Seriously. Fresh eyes tomorrow solve problems that 2 more hours won't.

The secret: Everyone hits these walls. The difference is knowing when to push through vs. when to step back and rethink.
